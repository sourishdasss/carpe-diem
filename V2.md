# Sonder — Claude Code Build Instructions (v2)

## What We're Building
A "Beli for Travel" **iOS app** where users rate individual attractions within cities, building a personal travel taste profile and a cumulative city score. Think Letterboxd meets Google Maps — personal, social, and opinionated.

## Core Concept
Instead of rating destinations as a whole, users rate **individual attractions** (museums, neighbourhoods, restaurants, viewpoints, experiences) within a city. Sonder then:
- Aggregates those attraction ratings into a **cumulative city score** (weighted by category)
- Infers a **personal travel taste profile** from rating patterns
- Surfaces a **social feed** of what people in your network have been experiencing
- Connects you to a **followers/following network** for social discovery

---

## Tech Stack
- **Platform**: iOS native (Swift + SwiftUI)
- **Minimum Deployment Target**: iOS 17+
- **AI**: Anthropic Claude API (`claude-sonnet-4-20250514`) via `URLSession` / `async await`
- **Images**: `AsyncImage` loading from Unsplash URLs (no API key needed)
- **Backend**: Supabase (PostgreSQL + Auth + Realtime)
- **Navigation**: SwiftUI `TabView` with a custom styled bottom tab bar (Feed / Lists / Profile)
- **Storage**: Supabase for persistence; `@StateObject` / `@EnvironmentObject` for local UI state

  summary text,

-- Attraction ratings
create table attraction_ratings (
  id uuid primary key default gen_random_uuid(),
  user_city_id uuid references user_cities(id) on delete cascade,
  attraction_name text not null,
  category text not null,
  rating int check (rating between 1 and 5),
  rated_at timestamptz default now()
);

-- Travel profile (AI-generated)
create table travel_profiles (
  id uuid primary key default gen_random_uuid(),
  user_id uuid references users(id) on delete cascade unique,
  personality_type text,
  personality_description text,
  taste_traits text[],
  updated_at timestamptz default now()
);

-- Photo pins (location-tagged photos uploaded by users)
create table photo_pins (
  id uuid primary key default gen_random_uuid(),
  user_id uuid references users(id) on delete cascade,
  user_city_id uuid references user_cities(id) on delete set null,
  photo_url text not null,           -- Supabase Storage public URL
  latitude double precision not null,
  longitude double precision not null,
  location_name text,                -- Reverse-geocoded place name (optional)
  caption text,
  taken_at timestamptz,              -- From EXIF DateTimeOriginal if available
  uploaded_at timestamptz default now()
);

-- Spatial index for fast bounding-box queries on the map
create index photo_pins_location_idx on photo_pins using gist (
  ll_to_earth(latitude, longitude)
);

-- Future travel recommendations (AI-generated from Supabase data)
create table travel_recommendations (
  id uuid primary key default gen_random_uuid(),
  user_id uuid references users(id) on delete cascade,
  destination text not null,
  match_reason text,
  vibe_tags text[],
  match_score numeric(4,2),
  generated_at timestamptz default now()
);

### FollowService.swift
```swift
import Supabase

class FollowService: ObservableObject {

    // Follow a user
    func follow(currentUserId: UUID, targetUserId: UUID) async throws {
        try await supabase
            .from("follows")
            .insert(["follower_id": currentUserId, "following_id": targetUserId])
            .execute()
    }

    // Unfollow a user
    func unfollow(currentUserId: UUID, targetUserId: UUID) async throws {
        try await supabase
            .from("follows")
            .delete()
            .eq("follower_id", value: currentUserId)
            .eq("following_id", value: targetUserId)
            .execute()
    }

    // Fetch follower count
    func followerCount(userId: UUID) async throws -> Int {
        let response = try await supabase
            .from("follows")
            .select("*", head: true, count: .exact)
            .eq("following_id", value: userId)
            .execute()
        return response.count ?? 0
    }

    // Fetch following count
    func followingCount(userId: UUID) async throws -> Int {
        let response = try await supabase
            .from("follows")
            .select("*", head: true, count: .exact)
            .eq("follower_id", value: userId)
            .execute()
        return response.count ?? 0
    }

    // Fetch list of followers (for display)
    func followers(userId: UUID) async throws -> [UserSummary] {
        let response: [FollowRow] = try await supabase
            .from("follows")
            .select("follower:users!follower_id(id, username, display_name, avatar_url)")
            .eq("following_id", value: userId)
            .execute()
            .value
        return response.compactMap { $0.follower }
    }

    // Fetch list of following
    func following(userId: UUID) async throws -> [UserSummary] {
        let response: [FollowRow] = try await supabase
            .from("follows")
            .select("following:users!following_id(id, username, display_name, avatar_url)")
            .eq("follower_id", value: userId)
            .execute()
            .value
        return response.compactMap { $0.following }
    }
}
```

## Future Travel Recommendations (Supabase-Powered)

### Overview
After a user has rated attractions across **3+ cities**, Sonder pulls their full rating history from Supabase and generates personalised travel recommendations via Claude. Results are stored back to the `travel_recommendations` table.

### How It Works

1. On Profile tab load, check if `travel_recommendations` has entries generated in the last 7 days
2. If stale or missing: fetch all `user_cities` + `attraction_ratings` for this user from Supabase
3. Send to Claude API — prompt is aware of categories, weights, and scores
4. Parse response, upsert results to `travel_recommendations` table
5. Display in `FutureRecommendationsView` on the Profile tab

### RecommendationsService.swift

```swift
class RecommendationsService {

    func refreshIfNeeded(userId: UUID) async throws -> [TravelRecommendation] {
        // 1. Check for recent recommendations
        let existing: [TravelRecommendation] = try await supabase
            .from("travel_recommendations")
            .select()
            .eq("user_id", value: userId)
            .gte("generated_at", value: ISO8601DateFormatter().string(from: Date().addingTimeInterval(-7 * 86400)))
            .execute()
            .value

        if !existing.isEmpty { return existing }

        // 2. Fetch rating data from Supabase
        let cities: [UserCityWithRatings] = try await supabase
            .from("user_cities")
            .select("*, attraction_ratings(*)")
            .eq("user_id", value: userId)
            .execute()
            .value

        guard cities.count >= 3 else { return [] }

        // 3. Call Claude
        let recommendations = try await ClaudeService.shared.generateRecommendations(from: cities)

        // 4. Upsert to Supabase
        try await supabase
            .from("travel_recommendations")
            .upsert(recommendations.map { rec in
                [
                    "user_id": userId.uuidString,
                    "destination": rec.destination,
                    "match_reason": rec.matchReason,
                    "vibe_tags": rec.vibeTags,
                    "match_score": rec.matchScore
                ]
            })
            .execute()

        return recommendations
    }
}
```

## Travel Map

### Overview
The **Map tab** shows a full-screen world map with a pin for every location a user has visited, derived from GPS metadata embedded in photos they upload. When a user uploads a photo with location data (EXIF GPS tags), Sonder extracts the coordinates, reverse-geocodes them to a place name, stores the result in the `photo_pins` Supabase table, and drops an animated pin on the map. Tapping a pin opens a popover showing the photo, place name, and date visited.

The map is the most visual tab in the app — it gives users an instant "at a glance" view of everywhere they've been in the world.

---

### How Location is Extracted from Photos

iOS photos can carry GPS coordinates in their EXIF metadata. Sonder reads this data at the point of upload using `PhotosUI` and `CoreLocation`. The flow is:

1. User taps **"+ Add Photo"** from the Map tab (or from within a city rating sheet)
2. `PHPickerViewController` presents the photo library — user selects one or more images
3. For each selected image, Sonder uses `CGImageSource` to read EXIF metadata and extract:
   - `{GPS}` dictionary → `Latitude`, `Longitude`, `LatitudeRef`, `LongitudeRef`
   - `{Exif}` dictionary → `DateTimeOriginal` (when the photo was taken)
4. If GPS data is present: coordinates are passed to `CLGeocoder` for reverse geocoding (city + country)
5. If GPS data is absent: user is prompted to manually pick the city from a search sheet
6. Photo is uploaded to **Supabase Storage** (`photos` bucket, path: `{userId}/{uuid}.jpg`)
7. A row is inserted into `photo_pins` with the coordinates, resolved place name, and storage URL
8. The map re-fetches the user's pins and drops the new one with a spring animation

> **Privacy note**: Location data never leaves the device during extraction. Only the resolved `latitude`, `longitude`, and `location_name` are stored in Supabase — raw EXIF data is discarded.

### PhotoLocationService.swift

```swift
import Photos
import PhotosUI
import CoreLocation
import ImageIO
import UIKit

struct PhotoPin: Identifiable, Codable {
    let id: UUID
    let userId: UUID
    let photoURL: String
    let latitude: Double
    let longitude: Double
    let locationName: String?
    let caption: String?
    let takenAt: Date?
    let uploadedAt: Date
}

class PhotoLocationService: ObservableObject {

    // MARK: - EXIF Extraction

    /// Extracts GPS coordinates from a UIImage's raw data via CGImageSource
    func extractCoordinates(from imageData: Data) -> CLLocationCoordinate2D? {
        guard
            let source = CGImageSourceCreateWithData(imageData as CFData, nil),
            let properties = CGImageSourceCopyPropertiesAtIndex(source, 0, nil) as? [String: Any],
            let gps = properties[kCGImagePropertyGPSDictionary as String] as? [String: Any],
            let lat = gps[kCGImagePropertyGPSLatitude as String] as? Double,
            let lon = gps[kCGImagePropertyGPSLongitude as String] as? Double,
            let latRef = gps[kCGImagePropertyGPSLatitudeRef as String] as? String,
            let lonRef = gps[kCGImagePropertyGPSLongitudeRef as String] as? String
        else { return nil }

        let finalLat = latRef == "S" ? -lat : lat
        let finalLon = lonRef == "W" ? -lon : lon
        return CLLocationCoordinate2D(latitude: finalLat, longitude: finalLon)
    }

    /// Extracts the DateTimeOriginal string from EXIF and converts to Date
    func extractDateTaken(from imageData: Data) -> Date? {
        guard
            let source = CGImageSourceCreateWithData(imageData as CFData, nil),
            let properties = CGImageSourceCopyPropertiesAtIndex(source, 0, nil) as? [String: Any],
            let exif = properties[kCGImagePropertyExifDictionary as String] as? [String: Any],
            let dateString = exif[kCGImagePropertyExifDateTimeOriginal as String] as? String
        else { return nil }

        let formatter = DateFormatter()
        formatter.dateFormat = "yyyy:MM:dd HH:mm:ss"
        return formatter.date(from: dateString)
    }

    // MARK: - Reverse Geocoding

    func reverseGeocode(coordinate: CLLocationCoordinate2D) async -> String? {
        let geocoder = CLGeocoder()
        let location = CLLocation(latitude: coordinate.latitude, longitude: coordinate.longitude)
        guard let placemarks = try? await geocoder.reverseGeocodeLocation(location),
              let placemark = placemarks.first else { return nil }

        let parts = [placemark.locality, placemark.country].compactMap { $0 }
        return parts.joined(separator: ", ")
    }

    // MARK: - Upload + Save Pin

    func uploadAndPin(
        imageData: Data,
        userId: UUID,
        userCityId: UUID? = nil,
        caption: String? = nil
    ) async throws -> PhotoPin {
        // 1. Extract location from EXIF
        let coordinate = extractCoordinates(from: imageData)
        let takenAt = extractDateTaken(from: imageData)

        guard let coordinate else {
            throw PhotoPinError.noLocationData
        }

        // 2. Reverse geocode
        let locationName = await reverseGeocode(coordinate: coordinate)

        // 3. Upload photo to Supabase Storage
        let photoId = UUID()
        let path = "\(userId)/\(photoId.uuidString).jpg"
        try await supabase.storage
            .from("photos")
            .upload(path, data: imageData, options: FileOptions(contentType: "image/jpeg"))

        let photoURL = supabase.storage.from("photos").getPublicURL(path: path).absoluteString

        // 4. Insert pin into photo_pins table
        let pin = PhotoPin(
            id: photoId,
            userId: userId,
            photoURL: photoURL,
            latitude: coordinate.latitude,
            longitude: coordinate.longitude,
            locationName: locationName,
            caption: caption,
            takenAt: takenAt,
            uploadedAt: Date()
        )

        try await supabase
            .from("photo_pins")
            .insert(pin)
            .execute()

        return pin
    }

    // MARK: - Fetch Pins

    func fetchPins(for userId: UUID) async throws -> [PhotoPin] {
        return try await supabase
            .from("photo_pins")
            .select()
            .eq("user_id", value: userId)
            .order("uploaded_at", ascending: false)
            .execute()
            .value
    }

    enum PhotoPinError: LocalizedError {
        case noLocationData
        var errorDescription: String? {
            "This photo doesn't have location data. Please select a city manually."
        }
    }
}
```

### TravelMapView.swift

The map uses `MapKit`'s `Map` view (iOS 17+ API). Each `PhotoPin` becomes a custom `MapAnnotation` rendered as a circular photo thumbnail with a terracotta ring. Tapping a pin presents a sheet with the full photo, location name, date, and caption.

```swift
import SwiftUI
import MapKit

struct TravelMapView: View {
    @StateObject private var service = PhotoLocationService()
    @State private var pins: [PhotoPin] = []
    @State private var selectedPin: PhotoPin? = nil
    @State private var showAddPhoto = false
    @State private var cameraPosition: MapCameraPosition = .automatic
    @EnvironmentObject var appStore: AppStore

    var body: some View {
        ZStack(alignment: .bottomTrailing) {
            Map(position: $cameraPosition) {
                ForEach(pins) { pin in
                    Annotation(
                        pin.locationName ?? "Visited",

                            latitude: pin.latitude,
                            longitude: pin.longitude

                        PinThumbnailView(pin: pin)
                            .onTapGesture { selectedPin = pin }
                    }
                }

            .mapStyle(.standard(elevation: .realistic))
            .ignoresSafeArea()

                showAddPhoto = true
            } label: {

                    .font(.system(size: 52))
                    .foregroundStyle(Color("Accent"))
                    .background(Color.white.clipShape(Circle()))
                    .shadow(color: .black.opacity(0.15), radius: 8, x: 0, y: 4)

                    .font(.custom("Georgia", size: 13))

            .padding(.leading, 16)

        .sheet(item: $selectedPin) { pin in
            PinDetailSheet(pin: pin)
        }
        .sheet(isPresented: $showAddPhoto) {
            AddPhotoSheet(onUpload: { newPin in
                withAnimation(.spring(duration: 0.4)) {
                    pins.insert(newPin, at: 0)
                }
                // Zoom to new pin
                cameraPosition = .region(MKCoordinateRegion(
                    center: CLLocationCoordinate2D(latitude: newPin.latitude, longitude: newPin.longitude),

                    longitudinalMeters: 80_000

            })
        }
        .task {
            pins = (try? await service.fetchPins(for: appStore.currentUserId)) ?? []
            if !pins.isEmpty {
                cameraPosition = .automatic
            }
        }
    }

    private func uniqueCountries(_ pins: [PhotoPin]) -> Int {
        Set(pins.compactMap { $0.locationName?.components(separatedBy: ", ").last }).count
    }
}

        path.addLine(to: CGPoint(x: rect.minX, y: rect.minY))
        path.addLine(to: CGPoint(x: rect.maxX, y: rect.minY))
        path.closeSubpath()
        return path
    }

            .onChange(of: selectedItem) { _, newItem in

                Task { await handleSelection(newItem) }
            }
            .toolbar {
                ToolbarItem(placement: .cancellationAction) {
                    Button("Cancel") { dismiss() }
                        .font(.custom("Georgia", size: 15))
                }
            }
        }
        .presentationDetents([.medium])
    }

    private func handleSelection(_ item: PhotosPickerItem) async {
        isUploading = true
        errorMessage = nil
        do {
            guard let data = try await item.loadTransferable(type: Data.self) else { return }
            let pin = try await service.uploadAndPin(
                imageData: data,
                userId: appStore.currentUserId,
                caption: caption.isEmpty ? nil : caption
            )
            await MainActor.run {
                onUpload(pin)
                dismiss()
            }
        } catch PhotoLocationService.PhotoPinError.noLocationData {
            errorMessage = "This photo has no location data. Try a different photo or add a city manually."
        } catch {
            errorMessage = "Upload failed: \(error.localizedDescription)"
        }
        isUploading = false
    }
}

## Attraction Rating System

When a user adds a city, they rate its individual attractions across categories. Each attraction gets a score of 1–5. The **cumulative city score** is calculated as:

- Weighted average across all rated attractions
- General appeal (walkability, vibe, safety, ease of getting around) counts as its own "attraction" with a weight of 1.5x
- Formula: `city_score = (sum of attraction scores × category weights) / total weighted count`, scaled to 10

### Category Weights
- Neighbourhoods & Walks: 1.5x (high signal for overall city vibe)
- Food & Markets: 1.2x
- Culture & Museums: 1.0x
- Viewpoints & Landmarks: 0.8x (often touristy, lower signal)
- Experiences & Activities: 1.0x

---

## Adding a City + Rating Attractions Flow

1. User taps **"+ Add City"** from the Lists tab
2. Searches for or selects a city
3. Presented with a pre-populated list of ~10 top attractions for that city (seeded from a static data file)
4. User rates each attraction they've visited (1–5 stars), skips ones they haven't
5. Also rates **General Appeal** (walkability, atmosphere, ease of navigation) on the same scale
6. On submit → Claude API call generates the cumulative score + a short AI-written city summary based on the ratings
7. City + ratings saved to Supabase (`user_cities` + `attraction_ratings`)
8. City appears in their Lists tab with the final score
9. If user now has 3+ cities, triggers a background recommendation refresh

## Claude API Usage

### 1. City Score + Summary Generation
After a user rates attractions in a city, send ratings to Claude using `URLSession` with `async/await`.

**System prompt:**
```
You are a travel analyst. Given a user's attraction ratings for a city, calculate a weighted cumulative score and write a personalized city summary. Respond in valid JSON only — no markdown, no preamble.
```

**User message format:**
```
The user visited [City, Country] and rated the following attractions:

...
General Appeal (walkability, vibe, atmosphere): X/5

Return this exact JSON shape:
